Android Setup

The following information is taken from the google help pages here. You should refer to them if anything isn't clear in the following instructions.

  1. In the Google APIs Console, create an API project for your application.
  2. In the Services pane, enable the Google+ API and any other APIs that your app requires.
  3. In the API Access pane, create an OAuth 2.0 client ID by clicking Create an OAuth 2.0 Client ID :
    • Type a product name in the dialog box that displays, and click Next. Providing a product logo and home page URL are optional.
    • Choose Installed application as your Application type and select Android as type.
    • In the Package name field, enter your Android's app's package name.
    • In a terminal, run the the Keytool utility to get the SHA-1 fingerprint of the certificate.

Getting the SHA1 fingerprint

In order to correctly sign your packaged application you will be using a p12 certificate. You'll need to extract the SHA1 signature of this certificate in order to complete the setup of the OAuth Client ID.

To do this you can use the keytool utility:

keytool -exportcert -keystore YOUR_CERTIFICATE.p12 -storepass YOUR_PASSWORD -list -v -storetype PKCS12

In order to correctly debug an AIR application in Flash Builder you have to setup your API access with the SHA1 signature from the AIR Debug Certificate. This is found in the location below.

keytool -exportcert -keystore /Applications/Adobe\ Flash\ Builder\ 4.7/eclipse/plugins/com.adobe.flexide.multiplatform.android_4.7.0.349722/resources/debug-certificate-android.p12 -storepass debug -list -v -storetype PKCS12

Manifest Additions

On Android you'll need to add the following to the manifest additions. 

     lt;uses-permission android:name="android.permission.INTERNET"/>
     lt;uses-permission android:name="android.permission.GET_ACCOUNTS" />
     lt;uses-permission android:name="android.permission.USE_CREDENTIALS" />
     
     lt;application>
     <meta-data android:name="com.google.android.gms.version" android:value="@integer/google_play_services_version" />
     <activity android:name="com.distriqt.extension.googleplus.activities.GooglePlusActivity" android:theme="@android:style/Theme.Translucent.NoTitleBar" />
     lt;/application>

iOS Setup

The following information is taken from the google help pages here. You should refer to them if anything isn't clear in the following instructions.

  1. In the Google APIs Console, select Create from the pull-down menu on the left, enter a project name (such as "iOS App"), and click the Create project button.
  2. In the Services pane, enable the Google+ API and any other APIs that your app requires by turning them on. If you plan to use other APIs, turn those on as well.
  3. In the API Access pane, click Create an OAuth 2.0 Client ID.
    • In the Product name field, enter a name for your application (such as "Sample"), and click Next. Providing a product logo is optional.
    • In the Create Client ID dialog box, shown below, do the following:
      • Select Installed application for the Application type.
      • Select iOS.
      • In the Bundle ID field, enter the bundle identifier from your application's project summary.
      • If your app is published in the Apple iTunes App Store, enter your app's App Store identifier in the App Store ID field. You can determine the ID from the "id" part of your app's URL on iTunes website: http://itunes.apple.com/us/app/id[YOUR_APP_STORE_ID]
      • Click the Create client ID button.
  4. In the API Access pane, locate the section Client ID for installed applications and note or copy the Client ID that you will need later.

Setting the bundle url

You'll need to add a URL type to your application and enter your bundle ID as the identifier and scheme. You do this by adding the following to the InfoAdditions node in your application descriptor, replacing YOUR_BUNDLE_ID with the Bundle ID you entered into the API Access pane previously. 

     <iPhone>
         <InfoAdditions><![CDATA[
             <key>UIDeviceFamily</key>
             <array>
                 <string>1</string>
             </array>
             
             <key>CFBundleURLTypes</key>
             <array>
                 <dict>
                     <key>CFBundleURLSchemes</key>
                     <array>
                         <string>YOUR_BUNDLE_ID</string>
                     </array>
                     <key>CFBundleURLName</key>
                     <string>YOUR_BUNDLE_ID</string>
                 </dict>
             </array>
         
         ]]></InfoAdditions>
         <Entitlements>
             <![CDATA[
             ]]>
         </Entitlements>
     </iPhone>

initialise()method 
public function initialise(clientId:String):Boolean

This function initialises the Google+ service. You should call this function once towards the beginning of your application.

The clientId should be the iOS client ID from your applications configuration in the Google developers console. Make sure you use the iOS application. The Android application is identified by the SHA1 fingerprint so the client id is not needed on Android.

Examples

The following example shows the simple process of initialisation and signing in a user 

     try
     {
         GooglePlus.init( APPLICATION_KEY );
         GooglePlus.service.addEventListener( GooglePlusEvent.SIGN_IN_SUCCESS,     signIn_successHandler, false, 0, true );
         GooglePlus.service.addEventListener( GooglePlusEvent.SIGN_IN_FAILED,     signIn_failedHandler, false, 0, true );
         GooglePlus.service.addEventListener( GooglePlusEvent.SIGN_OUT_SUCCESS,    signOut_successHandler, false, 0, true );
         GooglePlus.service.addEventListener( GooglePlusEvent.SIGN_OUT_FAILED,    signOut_failedHandler, false, 0, true );
     
         GooglePlus.service.initialise( CLIENT_ID );
         GooglePlus.service.signIn();
     }
     catch (e:Error)
     {
     }
     
     ...
     
     private function signIn_successHandler( event:GooglePlusEvent ):void
     {
         var person:Person = GooglePlus.service.getPerson();
         if (person != null)
         {
             trace( "Signed in as: " + person.displayName );
         }
     }

     {
     }

     }
